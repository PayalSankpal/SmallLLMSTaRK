ENTITY EXTRACTION TASK

OBJECTIVE: Extract entities from a natural language query and structure them in JSON format. One entity must be designated as "ANSWER", and others should use placeholder names (A, B, C, etc.).

EXTRACTION RULES

Rule 1: Entity Types
Only extract entities mentioned in the query.For each entity, provide a list of possible types from this list: disease, gene/protein, molecular_function, drug, pathway, anatomy, effect/phenotype, biological_process, cellular_component, exposure. An entity may have multiple types (e.g., "hypertension" can be both "disease" and "effect/phenotype").
If a general entity is mentioned in the query, such as "gene or protien" , extract it as an entity too and IF there is no semantic information for it, use the entire query as a semantic constraint for it, else use the semantic .
DO NOT EVER assign an entity a possible type not in this list.

Rule 2: Properties
If an entity has specific properties mentioned in the query, include them. Properties must be from this list: title, name.

Rule 3: Lexical Constraints
Identify hard lexical constraints where field values must match exactly. Example: for a disease node, "name" = "diabetes". Do NOT use general names like "skin disease" or "fever medicine". Use specific entity names only. for entities that do not have their name mentioned in the query, add the general information like "skin disease" in the semantic part.

Rule 4: Semantic Constraints
Provide a general description of the entity. Include ALL relevant information from the query. Can include descriptions, short phrases, and contextual information. Format as a list of descriptive strings.

Rule 5: Constant Field
Boolean field indicating whether the entity is a specific constant entity (e.g., "Ibuprofen", "CYP3A4") which should be true, or a general entity type (e.g., "some drug", "a disease") which should be false.

OUTPUT FORMAT

CRITICAL: Your response must be a valid JSON object with the following structure:

{
  "ENTITY_KEY": {
    "type": ["type1", "type2"],
    "lexical": {"property": "value"},
    "semantic": ["description1", "description2"],
    "constant": true/false
  }
}

FORMAT REQUIREMENTS: "type" is an array of strings. "lexical" is an object (can be empty {}). "semantic" is an array of strings (can be empty []). "constant" is a boolean (true or false, not a string).
